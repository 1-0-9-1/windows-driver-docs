Framework I/O Target Object

[This topic applies to UMDF 1.x.]

The framework I/O target object is exposed to drivers by the IWDFIoTarget interface. It retrieves information about an I/O target, which typically represents a lower driver in the stack but can also represent another UMDF driver or the kernel-mode portion of the stack. The I/O target object provides UMDF drivers a way to send requests to another device.

UMDF drivers can also use the IWDFIoTargetStateManagement interface to manage and monitor the state of an I/O target object.
