I/O Queue Event Callback Functions

[This topic applies to UMDF 1.x.]

When drivers create I/O queues, or configure default I/O queues, they can register the following interfaces so that the framework notifies the driver--by calling the methods associated with the interfaces--when events related to the interfaces occur. For more information about I/O queues and creating and configuring I/O queues, see Framework I/O Queue Object.

IQueueCallbackCreate

IQueueCallbackDeviceIoControl

IQueueCallbackRead

IQueueCallbackWrite

IQueueCallbackDefaultIoHandler
