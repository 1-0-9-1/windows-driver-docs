# Sample UMDF Drivers


This topic lists available User-Mode Driver Framework (UMDF) sample drivers that you can download from the [Windows driver samples repository](http://go.microsoft.com/fwlink/p/?LinkId=616507) on GitHub.

[Windows 8.1 driver samples](http://go.microsoft.com/fwlink/p/?LinkId=618052) can be downloaded from the [Windows Hardware Dev Center](http://go.microsoft.com/fwlink/p/?LinkId=616506).

## UMDF 2 Samples


-   [Sample Function Driver for OSR USB-FX2 (UMDF Version 2)](http://go.microsoft.com/fwlink/p/?LinkId=618003)
-   [Toaster Sample (UMDF Version 2)](http://go.microsoft.com/fwlink/p/?LinkId=618940)
-   [Echo Sample (UMDF Version 2)](http://go.microsoft.com/fwlink/p/?LinkId=617708)
-   [Power Framework (PoFx) Sample (UMDF Version 2)](http://go.microsoft.com/fwlink/p/?LinkId=617936)

## UMDF 1 Samples


-   [GPIO Sample Drivers](http://go.microsoft.com/fwlink/p/?LinkId=617729)
-   The HID client sample driver (Fx2Hid) sample was removed in Windows 8.1. If you are writing a Universal Windows app that communicates with a HID device, you'll use the Windows.Devices.Custom namespace to access the device's HID collections directly. For more information, see the [Custom driver access](http://go.microsoft.com/fwlink/p/?LinkId=618584) sample app and the [HidUsbFx2](http://go.microsoft.com/fwlink/p/?LinkId=618955) sample driver. If you are writing a Win32 application that accesses a HID collection, refer to the [HClient sample application](http://go.microsoft.com/fwlink/p/?LinkId=617730).
